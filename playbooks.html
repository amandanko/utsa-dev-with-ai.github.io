Tool Playbooks

Tool layer · dated & disposable

Playbooks are the perishable half of this curriculum: short, practical walkthroughs for specific tools, written to be replaced. Every playbook carries a last-verified date. If that date is more than a semester old, assume the screenshots are wrong, the pricing has changed, and a feature has been renamed — then apply Module 08's evaluation habit and submit a fix.

The playbook template

Every playbook follows the same six sections, so switching tools never means relearning the page. (This is also the structure students use when they write a new playbook as their Module 08 exercise.)

  1. What it is & which mode — inline assist, supervised agent, or autonomous, in one paragraph.
  2. Setup for this course — install, student licensing or free tier, and the permission settings we require (e.g., approval prompts on, auto-run off).
  3. Your first task — a single guided exercise using the course's sample project.
  4. Review workflow — where the diff lives, how to reject a change, how to see what commands the agent ran.
  5. Known sharp edges — current quirks, cost notes, and failure modes observed by students this semester.
  6. Verified-by line — who checked it, on what date, against which tool version.
Contributing

Found something outdated? Open an issue or a pull request on the course repository. Accepted student fixes are credited on the playbook's verified-by line — yes, that goes on a résumé.
